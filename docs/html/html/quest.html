handlers.quest.sign_request(params: dict, recvWindow: int, api_key: str, api_secret: str) → Tuple[list, dict]¶

Add signature to the request. Returns a list of params in tuples and a headers dict.

Parameters:

  • recvWindow (int or None) – Milliseconds of life for the request to be responded. /api/v3/historicalTrades do not admit it.

  • api_key (str) – Encoded API key.

  • api_secret (str) – Encoded API secret.

Return Tuple[list, dict]:

List of params in tuples and a headers dict.

handlers.quest.get_signed_request(url: str, decimal_mode: bool, api_key: str, api_secret: str, params: dict = None, recvWindow: int = 10000) → dict¶

It does a signed get to a url along with a dictionary of parameters. To avoid parameter order errors, they are passed in tuple format, it is also useful sending parameters that have the same name with several values:

https://dev.binance.vision/t/faq-signature-for-this-request-is-not-valid/176/4

Parameters:

  • url (str) – API endpoint.

  • decimal_mode (bool) – Sets numeric data format to decimal.

  • api_key (str) – Encoded API key.

  • api_secret (str) – Encoded API secret.

  • params (dict or List[tuple]) – Params for the request.

  • recvWindow (int) – Milliseconds of life for the request to be responded.

Return dict or list:

API response in dict or list format.
